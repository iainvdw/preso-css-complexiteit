    /**
 * Responsive table
 */
/* Slecht */
td { display: block; }

@media screen and ( min-width: 640px ) {
    td { display: table-cell; }
}

@media screen and ( max-width: 640px ) {

}

    BEM - Block, Element, Modifier

    /**
 * Product item component
 */
// Block
.product { [...] }

// Elements
.product__title { [...] }
.product__image { [...] }

// Modifiers
.product--highlighted { [...] }
.product__thumb--large { [...] }
    • Veelgehoorde opmerkingen:
    • Lelijk
    • Te lange classnames
    • Dubbele underscores en dashes in 1 classname

    Blocks, Elements, Modifiers

    Beschrijving van product

    • Blocks zijn containers, het startpunt en afgrenzing van een component
    • Denk aan Blocks als Lego blokjes, meerdere Blocks bij elkaar vormen een groter geheel
    • Elements zijn children van Blocks
    • Modifiers zijn toepasbaar op Blocks en Elements
    • Modifiers zijn lichte varianten in presentatie

    Voordelen van BEM

    • Verdeelt styling onder componenten
    • Uniforme naamgeving over hele codebase
    • Relaties tussen elementen af te leiden uit HTML
    • Lage specificty
    • Moedigt aan tot ontleden van een ontwerp tot kleine herbruikbare componenten
    • Classnames in een stylesheet zijn uniform en uniek over alle componenten
    • Niet tot nauwelijks nesten selectors

    OOCSS

    Object-oriented CSS

    OOCSS — principes

    1. Separate structure from skin
    2. Separate containers from content

    Separate structure
    from skin

    .btn-primary {
  border: 2px solid orangered;
  border-radius: 3px;
  padding: 6px 12px;
  font-size: 1.2rem;
  font-weight: bold;
  background: orange;
}
    		 
    .btn-default {
  border: 1px solid limegreen;
  border-radius: 3px;
  padding: 5px 10px;
  font-size: 1rem;
  font-weight: normal;
  background: lime;
}
    • Classes kunnen uit visuele en structurele properties bestaan
    • Deze groepen kunnen van elkaar gescheiden worden tbv hergebruik op andere plekken

    Separate structure
    from skin

    /* Structure */
.btn-medium {
  border: 1px solid;
  padding: 5px 10px;
}

.btn-large {
  border: 2px solid;
  padding: 6px 12px;
}


    /* Skin */
.btn-primary {
  border-color: orangered;
  font-size: 1.2rem;
  font-weight: bold;
  background-color: orange;
}

.btn-default {
  border-color: limegreen;
  font-size: 1rem;
  font-weight: normal;
  background-color: lime;
}
    • Scheid type properties van elkaar
    • Mix 'n match classes met elkaar voor extra variatie
    • Structuur en uiterlijk zijn gescheiden

    Separate containers
    from content

    /* Slecht */
.page h1 {
  [...]
}

/* Beter */
.page-title {
  [...]
}
    • Zorg dat styling niet afhankelijk is
      van de locatie in je HTML
    • Houdt elementen zonder class hetzelfde en elementen met class krijgen de stijling
    • Bevordert hergebruik van classes in je project
    • Zorgt ervoor dat je later geen properties hoeft te
      unsetten op elementen waar je de styling niet wilt

    OOCSS Mixins

    /* Structure */
@mixin btn-medium {
  border: 1px solid;
  padding: 5px 10px;
}

@mixin btn-large {
  border: 2px solid;
  padding: 6px 12px;
}


    /* Skin */
@mixin btn-primary {
  border-color: orangered;
  font-size: 1.2rem;
  font-weight: bold;
  background-color: orange;
}

@mixin btn-default {
  border-color: limegreen;
  font-size: 1rem;
  font-weight: normal;
  background-color: lime;
}
    		 
    .btn-cta {
  @include btn-large;
  @include btn-primary;
}
    • Verplaats losse OOCSS classes naar mixins
    • Stel classes samen obv deze mixins
    • Houdt je HTML en CSS stricter gescheiden, minder tot geen class clutter
    • Geen meerdere classes nodig, alle styling zit in 1 class
    • Geeft je de mogelijkheid beschrijvende namen te geven aan je class ipv generieke namen als .btn.btn-primary.btn-large

    OOCSS Mixins
    extra voordelen

    • Alle styling zit in 1 selector
    • Gzip maakt korte metten met gedupliceerde properties, output is zelfs kleiner dan via @extend
    • Simpliciteit van output is vele malen hoger
    • Ongebruikte mixins genereren ook geen output meer
