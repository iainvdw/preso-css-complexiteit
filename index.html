    BEM

    Block, Element, Modifier

    
.article-list { ... }
.article-list__item { ... }
.article-list__item--highlighted { ... }

    Wat is BEM?

    • Scheidt je pagina in componenten
    • De componenten bestaan uit Blocks en Elements
    • Blocks zijn containers
    • Elements zijn children
    • Directe relatie tussen Block en Element in class name

    Waarom BEM?

    • Moedigt aan tot nadenken over unieke componenten en relaties tussen elementen
    • Uniforme naamgeving over hele codebase
    • Relaties tussen elementen afleiden uit HTML
    • Niet tot nauwelijks nesten selectors
    • Uitermate geschikt voor doorontwikkeling op lange termijn

    OOCSS

    • Separate structure from skin
    • Separate containers from content

    Separate structure from skin

    • Trek visuele (font, kleur, border) los van structuur (display, margin, padding)
    • Stel componenten samen op basis van deze abstracties
    • Visuele abstracties kunnen gebruikt worden als modifiers bovenop structuur
    • Single responsibility principle
    • Separation of concerns

    Separate containers from content

    • Zorg dat styling niet afhankelijk is
      van de locatie in je HTML
    • Houdt elementen zonder class hetzelfde en elementen met class krijgen de stijling
    • Bevordert hergebruik van classes in je project
    • Zorgt ervoor dat je later geen properties hoeft te
      unsetten op het moment dat je de styling toch niet wilt
